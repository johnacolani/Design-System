Principles

Decisions locked for ServiceFlow multi-tenant SaaS UI.

  • Tetradic anchors — Primary (teal) #6FA8A1 · Coral (CTA / highlights) #D98C7A (complement to teal). Secondary (purple) #7B6F9D · Gold (soft accent) #C9A96E (complement to purple). Ramps use HSL lightness steps (±0.04 per step) from each anchor.
  • Pairing — Teal ↔ coral for action vs. brand structure; purple ↔ gold for secondary surfaces and premium hints. Use coral for primary filled buttons; teal for links, focus rings, and selection states.
  • Surfaces — Light app background #F5F2EB. Dark scaffold #1A1917 (warm charcoal).
  • Staff dashboards — Deep teal–purple gradient headers and body washes; each role gets a distinct mix of the same tetrad via ServiceFlowDashboardTheme.
  • Type — Roboto with system-ui fallbacks; scale uses px sizes documented in ServiceFlowTypography.
  • Grid — 12 columns, 16px gutter (ServiceFlowSpacing).

Alert dialogs (Android / web vs iOS)

In Flutter, showPlatformDialog in lib/core/widgets/platform_dialog.dart uses AlertDialog (Material) on Android and web, and CupertinoAlertDialog inside showCupertinoDialog on iOS. Destructive actions map to red text on both; default iOS action is bold. iOS dialogs are wrapped in CupertinoThemeData(brightness: Brightness.light) so they stay readable when the app uses a dark scaffold.

Material · showDialog + AlertDialog

Actions in code: non-destructive → ElevatedButton, destructive → TextButton with red label (map Material primary to teal / coral tokens if desired).
